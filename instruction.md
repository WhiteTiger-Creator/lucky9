Compute the Meridian-2 flux routing for a directed weighted network and report the routed flux.

Flux routing is a combinatorial network problem: from a single source you route flux along directed pathways, and the key quantity is how much flux can be pushed out of the source along **node-disjoint** pathways at once — pathways that compete for shared nodes cannot all be used, so the routable flux is a maximum-weight node-disjoint packing of pathways, deliberately **not** the sum of each reachable node's best pathway (that over-counts shared nodes). `/app/docs/model_spec.md` defines the network format, the canonicalization rules, the pathway definition and hop bound, the routed-flux objective, and the exact output schema and checksum serialization. Read it closely — the objective is a packing, and the natural per-target sum is a wrong answer.

Given the network in `/app/data/network.json`, compute the routing and record the results in `result.json`: the reachable nodes, the strongest single pathway and its weight, the routed flux `max_flux`, and the `edge_checksum` and `flux_checksum` over the exact serializations the spec defines.

Implement the computation as `/app/flux.py`, reading `--input` (default `/app/data/network.json`) and writing under `--output-dir` (default `/app/output`), and run it so `/app/output/result.json` is in place. Derive every value from the input; don't read or import anything from `/tests` or `/solution`.
